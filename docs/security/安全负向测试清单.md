# Context Engine — 安全负向测试清单

## Core Idea

> 安全验收以 Unauthorized Evidence = 0、wrong-Organization effect = 0 和 missing-context fallback = 0 为硬门槛；授权前的 CandidateRef 不含正文，Runtime 内只有 AuthorizedProjection 可以进入content-bearing rerank、扩展与Assembler；BotDelivery 的 ModelGateway 只接收从一个当前、audience-bound ContextPackage派生的 AuthorizedModelInput 与匹配 EgressGrant。每个攻击面同时映射到 domain oracle、真实 Postgres/Adapter enforcement、Runtime Package 与 audit redaction，单层绿灯不构成通过。

## Content

ContextEngine 的负向测试 catalog 从自有威胁模型与 release contract 推导，把租户越权、缺失上下文、证据泄漏、授权时序、工作者身份、audience drift、egress 与外部 effect 分离转成稳定 test IDs。[ADR-0019](../decisions/0019-security-catalog-normalization.md) 将发布权威归一为恰好 15 个稳定 ID；机器权威是 [`eval/catalogs/security-invariants.yaml`](../../eval/catalogs/security-invariants.yaml)，由 [`eval/catalogs/security-catalog.schema.json`](../../eval/catalogs/security-catalog.schema.json) 和 `python3 scripts/validate_security_catalog.py` 校验。目录采用 JSON-compatible YAML，使 D0 在尚无依赖清单时可由 Python 标准库读取。四个公开参考仓只用于校验 Adapter、parser/retrieval、preview/release 和真实依赖测试的工程形状；不继承其安全保证，也不声称已运行上游渗透、故障注入或 ContextEngine 动态 Spike。一手证据和未取证边界见[Four Public Repositories Evidence Baseline](../research/2026-07-19-four-public-repositories-evidence.md)。

这里的“无 mock 能力声明”是证据门槛，不是禁止测试替身。deterministic fixture、fake、twin、spy 与 property generator 可以证明内部失败语义和“调用/bytes/effect = 0”；只有真实 PostgreSQL、真实 wire、真实 source sandbox 或真实外部依赖运行结果才能支持 live capability claim。

测试接受两个代价：PR 必须运行真实 PostgreSQL 17 non-owner RLS suite，速度高于纯 unit；denied/not-found non-enumeration 需要 response/timing 归一化，debug 便利度下降。security gate 不与 retrieval quality、latency 或总分平均。

> **一个 unauthorized Fragment 即使最终没出现在答案里，也已经是 Context Runtime failure。**

[ADR-0019](../decisions/0019-security-catalog-normalization.md) 只归并发布 ID，不删除任何 safeguard 或 case：原 `AUDIENCE-016` 全部语义映射到 `SCOPE-INTERSECTION-004` + `EGRESS-011`；原 `ACL-PROOF-017` 映射到 `INDEX-NOT-AUTHORITY-005` + `REVOCATION-006`；原 `DELIVERY-EVIDENCE-018` 映射到 `TRANSPORT-UNTRUSTED-008`。下文对应的 `AUTH-*`、`DELIV-*`、`PROV-*`、`RUN-*`、`IM-*` 等案例继续执行，并在机器目录的 canonical invariant 下提供证据，不形成第 16–18 个发布 ID。

`CACHE-SCOPE-013` 保留为 canonical 15 之外的预注册条件扩展：当前必须用 composition/behavior case 证明不存在 authorization-sensitive 的最终 `ContextPackage` 或 `AuthorizedProjection` cache。首次激活这类 cache 时，必须先通过版本化 catalog/schema 变更加入 `CACHE-SCOPE-013` 及其 key/envelope 绑定完整 identity、policy、config、`asOf` 的 mutation cases；只有 `PASS` 才能发布。既有编号不重排、不复用。

[Test Architecture and Verifiability Design](./Test-Architecture-与可验证性设计.md) 定义 portfolio；这份文档只保存安全 case 与精确 oracle。

每条 invariant 分开记录三个维度：在版本化 catalog 中预注册的 applicability（required、带`applicableFrom`的conditional，或带审批理由的NOT_APPLICABLE）、capability activation/coverage（unavailable、implemented、contract-verified、sandbox-verified、live-verified），以及 applicable active path的当次 PASS/FAIL结果。对外渲染态仍只能取 `PASS`、`FAIL`、`NOT_ACTIVE`、`NOT_APPLICABLE`。`NOT_APPLICABLE`必须在运行前由冻结 catalog 排除；`NOT_ACTIVE`必须证明该边界不可达；active但未运行、未映射或缺证据一律是`FAIL`。每个 milestone 的 required exit 只能由`PASS`满足，`NOT_ACTIVE`和`NOT_APPLICABLE`都不能替代。

> **Invariant 是长期名称，具体 fixture可以演化；release report必须逐条给出 PASS/FAIL/NOT_ACTIVE/NOT_APPLICABLE 与证据。**

property generator 随机创建 Organization、Membership、Group、Principal grant、Agent ceiling、Source ACL、Resource ACL、Purpose policy与optional RequestNarrowing，验证 trusted operands交集的交换律、空集、单调性与Oracle/DB一致；另外验证RequestNarrowing缺省是相对已建立trusted scope的identity element，不是空集、全局scope或可选的trusted authorization operand。

`TrustedDeliveryContext` 是受信入口构造的nominal调用事实，至少绑定 Organization、requesting Principal、delivery target、private/group audience kind、purpose 与 AudienceSnapshot ref/digest/asOf。BotDelivery可以向trusted identity Adapter请求一个短时DeliveryEvidenceRef，但在远程wire上只能通过认证transport metadata传该opaque ref；body中不得携带raw trusted identity、purpose 或 audience claims。Ingress必须校验authenticated service、resolve request id、Organization、asker、destination、purpose、audience digest和expiry后才兑换为TrustedDeliveryContext。AudienceSnapshot只含事实，群scope由Kernel计算。

> **身份错误在检索前失败；资源授权与字段投影发生在任何content-bearing rerank/model之前；两者都不能由LLM处理。**

| `CACHE-002` | cache命中旧Policy Epoch Package | validate失败；重新acquire；旧content不返回 |
| `BLOB-001` | 猜object path/换org prefix/复用signed URL | denied/not-found等价；bytes=0 |

request与worker operation必须各自使用一个明确事务，并在任何ORM autoflush、query或provider/retrieval工作之前transaction-locally绑定并校验Organization与closed `ActorContext`。在线request使用含authenticated Principal/Membership的`UserActor`；Supply worker使用注册、最小权限的`ServiceActor`，不得伪装触发User，也不得把ingestion authority当作最终用户delivery authority。其WorkerLease绑定Organization、job、operation、source、可选resource/revision、ServiceActor/workload、policy epoch、可选audience、idempotency、lease generation、iat/exp与nonce，兑换时逐项核对durable job row。runtime/worker role必须是non-owner、`NOSUPERUSER NOBYPASSRLS NOINHERIT`，tenant table必须`FORCE ROW LEVEL SECURITY`。schema security manifest必须把**每一张表**分类为global或tenant，而不是只统计已经有`organization_id`的表；tenant分类要求organization key、composite ownership约束、RLS policy与负向fixture。

对 worker 实施 missing-tenant 回落、对 index/cache/blob 实施租户过滤但不做 exact authorization，都是 ContextEngine 自有威胁模型要杀死的隐含前提。因此 `DB-001/002`、worker lease、cache-key mutation 与 blob-open 测试都必须默认拒绝，不允许任何 default organization/public scope。

> **Index/cache/blob可以延迟清理，任何一个都不能绕过current policy决定Evidence与bytes。**

`PROV-012` 从“普通 Membership 可枚举 connector 但不得读取 credential”这一 ContextEngine 边界直接推导。目标实现将 credential ref 与 public config 分型，Transport schema 不包含 secret；验收必须使用真实 schema/HTTP response 与日志扫描，不由 fake Adapter 单独开门。

`ContextProvider` V1 仅有四个typed只读操作：`describeCapabilities`、`readChanges`、`discover`、`authorizeAndProject`，且全部返回closed `ProviderOutcome = Ok | Unsupported(capability) | RetryableUnavailable(retryAfter) | InvalidCheckpoint | GenericDenied`。`discover`只返回content-free CandidateRef；`authorizeAndProject`返回的SourceProjectionBatch是给Kernel校验的证据，不是caller可直接信任的AuthorizedProjection。CandidatePage与SourceProjectionBatch共享绑定provider、SourceVersion、ACL mode、decision/snapshot与asOf的SourceConsistencyRef；缺失、混用、变化或过期时Kernel拒绝。FileProvider使用PostgreSQL中显式、版本化且随File Source激活的FileSourceAccess作为Mirrored SourceAclEvidence，不声称镜像宿主OS文件权限。缺失、不完整或未知grant一律deny，不推断implicit owner或public。

ACL proof只有三种合法语义：`live`是在同一resolve中向source按当前subject做授权；`mirrored`是对明确`aclAsOf/sourceVersion`与SLA内快照的精确授权；`weak`仅适用于source原生就没有更细ACL的coarse membership场景，并必须暴露更弱保证和敏感内容deny策略。strong Provider观测失败不能降级为weak；只能fail closed、返回generic unavailable gap或使用另一个同等/更强的proof。

> **Provider拥有最新业务对象与字段权限；service account只允许它执行检查，不替代最终User授权。**

| `RUN-006` | Policy Epoch变化后continue | stale continuation拒绝；新resolve重新决策 |
| `RUN-007` | multi-hop continuation或denied candidate mint offer | 每个child scope是parent子集；denied target offer数=0 |
| `RUN-008` | budget字段省略/增大/多跳重复消费 | token/provider call/cost按remaining累计且不超额；latency只按per-call ceiling取min |
| `RUN-009` | wrong Runtime consumer/session audience兑换 | generic unavailable；provider/index调用0 |
| `RUN-010` | 同token sequential/concurrent replay | V1仅一次成功与一次effect；其余generic unavailable |
| `RUN-011` | epoch未变，live ACL/freshness/budget改变 | 成功返回self-contained replacement；旧Evidence与dangling refs为0 |
| `RUN-012` | unknown kind、need/continuation混合或trusted字段注入 | closed wire schema拒绝；HTTP与generated SDK一致，MCP激活后须同一canonical error |
| `RUN-013` | CandidateRef/SourceProjectionBatch/denied data被传给Runtime content rerank或Assembler，或AuthorizedProjection直接传给ModelGateway | type/import/spy boundary拒绝；Runtime consumer只收AuthorizedProjection，ModelGateway只收AuthorizedModelInput |
| `RUN-014` | 同一问题需要群公开结果与提问者私有补充 | 两次独立audience-bound resolve；public Package不从asker Package事后分割 |
| `RUN-015` | BotDelivery用两个Package、stale/wrong-audience Package、raw provider content或denied data构造model input | AuthorizedModelInput构造失败；model request bytes=0 |
| `CITE-001` | Package外伪造EvidenceRef/openRef | not available；source/blob call在授权后或0 |
| `CITE-002` | valid citation在Membership revoke后open | bytes/fields=0；response等价not-found |
| `CITE-003` | CitationOpenRef作为ContinuationToken兑换，或反向兑换 | wire variant/audience验证拒绝；provider/index调用0 |
| `CITE-004` | 两名群成员依次打开同一CitationOpenRef | 每次认证opener并exact authorize；允许的成员各自成功，denied成员bytes=0；ref本身不授予权限 |
| `EGR-001` | source sensitivity与model/provider/region不兼容 | provider payload=0；Package gap generic restricted |
| `EGR-002` | prompt/debug/trace序列化在filter前执行 | test捕获secret/denied content并失败 |
| `EGR-003` | BotDelivery绕过Kernel、自行扩大AudienceSnapshot scope或从asker Package切public内容 | public resolve/send effect=0；audit记录trusted-boundary violation |
| `EGR-004` | ModelGateway无匹配purpose/provider/region/audience EgressGrant，或收到非AuthorizedModelInput | model request=0；不得从ContextPackage自行推导授权 |
| `EGR-005` | 群成员在resolve后、send/finalize前退出/加入，或member lookup超时 | Sender重新验证；stale/unknown时群bytes=0并可转private |
| `EGR-006` | 新成员可读取群历史，而Package包含旧snapshot非全员可见内容 | public send被策略拒绝，或使用可验证删除/redaction/private delivery；future-member bytes=0 |
| `ACTION-001` | ContextAccessTicket调用refund/update/send | wrong audience拒绝；business effect=0 |
| `ACTION-002` | ActionTicket调用`resolve(kind=acquire)`/provider read | wrong capability拒绝；Evidence=0 |
| `ACTION-003` | `prepare(CreatePlaceholder intent)`签发的ticket用于`perform(FinalizeReply payload)`，或反向 | effect/action/audience mismatch拒绝；business effect=0 |
| `ACTION-004` | create与finalize分别经prepare获得的ticket被sequential/concurrent replay | 每张one-shot ticket最多一次对应effect；已成功replay只返回`AlreadyApplied`与存量receipt |
| `ACTION-005` | 绕过prepare直接调perform，或caller自签ActionTicket | ticket validation失败；Sender call和business effect=0 |
| `ACTION-006` | perform的payload digest、destination、audience、epoch、expiry、approval tier或idempotency key与prepare绑定不同 | exact binding拒绝；wrong-Organization/wrong-audience effect=0 |
| `ACTION-007` | provider调用结果ambiguous/timeout-after-send，caller尝试换新ticket或idempotency key重发 | 返回`ReconciliationRequired`；保持同一attempt/id对账；新ticket和额外effect=0 |
| `ACTION-008` | prepare安全拒绝、audience drift或暂时不可用 | 仅返`GenericDenied`、`AudienceChanged`或`RetryableUnavailable`；ActionTicket/Sender effect=0 |
| `ACTION-009` | perform拒绝wrong ticket/payload或经对账才能确定的external attempt | 仅返`Rejected(effect=0)`或`ReconciliationRequired`；不将不确定状态伪装成retryable新effect |

`ContinuationToken`绑定principal/audience/purpose/scope/remaining budget，one-shot兑换replacement Package；`CitationOpenRef`只是可重复使用的opaque locator，不包含授权能力，每次open都认证opener并执行current exact auth。两者使用不同wire variant、audience、key purpose与audit event。

Trusted ingress、BotDelivery、ModelGateway、ActionPlane与Sender组成delivery TCB。Runtime内部的content-bearing rerank、hydration、relevance model与Assembler只能消费AuthorizedProjection；下游生成边界不接收该类型。BotDelivery只能从**一个**当前、audience-bound ContextPackage与其purpose/retention policy构造AuthorizedModelInput，ModelGateway只在同时收到匹配model EgressGrant时发出bytes。外部写效果只经`ActionPlane.prepare(TrustedEffectIntent) -> ActionPreparationOutcome`后的`ActionPlane.perform(EffectPayload, ActionTicket) -> ActionExecutionOutcome`；prepare只返回`Prepared | GenericDenied | AudienceChanged | RetryableUnavailable`，perform只返回`Applied | AlreadyApplied | Rejected(effect=0) | ReconciliationRequired`。每个effect独立绑定operation、destination、audience、payload digest、epoch、expiry、approval tier与idempotency key；已成功replay只返回存量receipt，ambiguous provider attempt必须使用同一attempt/id对账，不换新ticket。Sender在最终发送前还必须重新验证AudienceSnapshot。

ContextPackage snapshot同时检查 purpose、TTL、asOf、decisionRef、Policy Epoch、freshness、trust、coverage/gaps与authorized citation。HTTP与generated SDK canonicalize后等价；MCP只在真实caller激活后加入同一conformance suite。Transport不能删除security fields或添加自己的ACL逻辑。

> **Runtime assembly只接收AuthorizedProjection；ModelGateway只接收BotDelivery构造的AuthorizedModelInput与匹配EgressGrant；两个nominal type不可互换。**

| Test ID | Leak/mutation | Expected |
|---|---|---|
| `OBS-001` | denied Resource名称/content/score/count进入trace | snapshot/redaction test失败；只留opaque ref与policy category |
| `OBS-002` | connector credential/token/AccessTicket/WorkerLease/ContinuationToken写log或error | raw secret match=0；必要claims只写digest |

| `ENUM-002` | denied/missing latency分布比较 | 同coarse bucket；超threshold进入security finding |

跨租户学习默认关闭。允许的长期方向只有opt-in、聚合、匿名化、可审计与可删除的artifact；任何 pattern-learning 方案都必须从这一默认拒绝规则开始，不得继承旧研究中的放宽假设。

`ContextRun`是tenant-visible/learning-safe记录，只能引用AuthorizedProjection产生的Evidence与Package digest；`DecisionAudit`是访问、保留、导出都独立受控的security record，只保存opaque candidate ref/digest、policy rule与denial category，不保存raw denied正文。ContextControl、ContextRuntime、ContextLearning、BotDelivery与ActionPlane的release责任不重叠：ContextControl只管source/access/policy，ContextRuntime只解析Package，ContextLearning用`evaluate`生成评测结果并通过唯一`promote`入口激活/回滚ReleaseManifest，BotDelivery编排交付，ActionPlane独占外部effect。ReleaseManifest中的CurationProfile必须显式表示curation-off，或引用一个CurationSnapshotRef及兼容Revision集与评测digest。

> **审计需要知道系统为何拒绝，caller与跨租户学习管道没有知道被拒内容的需要。**

任何 community/enterprise、shared/dedicated 或 feature-flag 变体都按实际可取证 Adapter/edition 运行同一 contract suite，不能从 UI、README、同名 capability 或另一 edition 继承结果。不在当前 checkout/sandbox 中可动态验证的能力必须标为 `NOT_ACTIVE`、`NOT_APPLICABLE` 或 `FAIL`，不能被写成 `PASS`。

> **edition与placement可以改变Implementation，不能改变同一Context contract的安全下限。**

### 9. Minimal release matrix

同一句query、同一Agent的固定矩阵：

| Principal context | File Source | Ticket Source fields | Expected |
|---|---|---|---|
| Org A / owner | A file | id/status/customer/private_note | A完整Package |
| Org A / limited | A file按grant | id/status；无private_note | A受限Package |
| Org B / owner | B同名file | B ticket | B完整Package，与A Evidence互斥 |
| Org A / revoked | none | none | empty/generic；旧cache/continuation/citation失效 |
| missing Membership | no acquisition | no provider call | invocation fail closed |

矩阵通过HTTP、generated SDK、worker replay、malicious candidate injection、federated authorizeAndProject、continue/open等已激活path；MCP在真实caller出现后再加入同一矩阵。任何active path出现unauthorized Evidence或wrong-org effect，整个release失败。

另加群交付fixture：A/B同群且都能看公开证据，A另有private field；公开resolve绑定AudienceSnapshot `{A,B}`，private resolve仅绑定A。B、未来加入成员C、发送前身份解析失败三种情况都不能收到A的private bytes；不得先按A resolve再由Gateway切分。

File fixture中的每个allowed result都必须有当前激活FileSourceAccess的完整Mirrored ACL证据；missing/incomplete/unknown grant与denied返回同类generic结果，owner身份、宿主文件权限或“默认public”都不得填补缺失grant。

Visible self-correction：早期清单把“跨租户query返回不同答案”当主要oracle。答案由LLM生成，可能偶然省略越权内容。修正后，核心oracle前移到 AuthorizedProjection/Evidence/Provider payload/side effect；answer-level检查只做补充。

> **最终答案可以不稳定，授权后的Evidence集合与side effect必须确定。**

### 10. Execution cadence

| Cadence | Suites |
|---|---|
| Every commit | property、real Postgres RLS/FK/outbox、Adapter contracts、Module behavior、core security matrix、capability status evidence |
| Retrieval-sensitive PR | frozen retrieval/assembly + all security cases |
| Nightly | fuzz、fault injection、live Provider conformance、timing non-enumeration、large corpus |
| Release candidate | packaged image、migrate、non-owner role、two-Org fixture、HTTP/generated SDK、worker/citation/action；MCP激活后再列入 |
| Engineering Gate E5 | Security/Reliability/Quality/Budget required catalog条目全部PASS + Ops readiness；不以partner存在为前提 |
| Launch Gate L1 | E5后单独验证design-partner agreement、legal review、命名与commercial approval；通过前不开放受邀使用 |
| Incident/finding | add minimal reproducer to stable invariant ID before fix；rerun full related family |

Flaky deterministic security/reliability case不能自动重跑变绿。所有报告固定 commit、schema、Adapter/config/model/index/dataset版本与raw result digest，历史分数不跨baseline继承。

> **安全catalog随新攻击面追加，Invariant ID保持稳定，便于审计每次release实际证明了什么。**
